  Overview
  We will create a self-contained MSSQL Docker image that initializes the database schema and seeds it with the necessary catalog and author data required for functional testing. This ensures your tests
  run against a consistent, pre-populated environment.

  Step 1: Prepare the Setup Script
  Create a new file src/Hmm.Docker/setup.sql. This file combines the schema creation and data seeding into one execution flow. We need to handle IDENTITY_INSERT settings for MSSQL since the seed scripts
  include explicit IDs.

  File: src/Hmm.Docker/setup.sql

    1 /* 1. Create Database and Schema */
    2 :r /init-db.sql
    3
    4 /* 2. Seed Data */
    5 USE hmm;
    6 GO
    7
    8 /* Enable Identity Insert for tables we are about to seed */
    9 SET IDENTITY_INSERT dbo.Authors ON;
   10 SET IDENTITY_INSERT dbo.NoteCatalogs ON;
   11 SET IDENTITY_INSERT dbo.NoteRenders ON;
   12 SET IDENTITY_INSERT dbo.Subsystems ON;
   13 GO
   14
   15 /* Run the seeding script */
   16 :r /seed-data.sql
   17 GO
   18
   19 /* Disable Identity Insert */
   20 SET IDENTITY_INSERT dbo.Authors OFF;
   21 SET IDENTITY_INSERT dbo.NoteCatalogs OFF;
   22 SET IDENTITY_INSERT dbo.NoteRenders OFF;
   23 SET IDENTITY_INSERT dbo.Subsystems OFF;
   24 GO

  Step 2: Create the Entrypoint Script
  Create a shell script src/Hmm.Docker/entrypoint.sh to orchestrate the startup. It will start the SQL Server in the background, wait for it to be ready, run our setup script, and then keep the server
  running.

  File: src/Hmm.Docker/entrypoint.sh

    1 #!/bin/bash
    2
    3 # Start SQL Server in the background
    4 /opt/mssql/bin/sqlservr &
    5 pid=$!
    6
    7 # Wait for SQL Server to start
    8 echo "Waiting for SQL Server to be ready..."
    9 for i in {1..60};
   10 do
   11     /opt/mssql-tools18/bin/sqlcmd -S localhost -U sa -P "$MSSQL_SA_PASSWORD" -C -Q "SELECT 1" > /dev/null 2>&1
   12     if [ $? -eq 0 ]
   13     then
   14         echo "SQL Server is ready."
   15         break
   16     else
   17         echo "Not ready yet..."
   18         sleep 1
   19     fi
   20 done
   21
   22 # Run the setup script
   23 echo "Running setup script..."
   24 /opt/mssql-tools18/bin/sqlcmd -S localhost -U sa -P "$MSSQL_SA_PASSWORD" -C -i /setup.sql
   25
   26 # Wait for the SQL Server process to finish (which should be never, unless stopped)
   27 wait $pid

  Step 3: Update the Dockerfile
  Update the existing src/Hmm.Docker/DockerFile-db (or create src/Hmm.Docker/Dockerfile) to package everything.

  File: src/Hmm.Docker/Dockerfile

    1 FROM mcr.microsoft.com/mssql/server:2022-latest
    2
    3 USER root
    4
    5 # Install dos2unix to fix potential line ending issues with scripts from Windows
    6 RUN apt-get update && apt-get install -y dos2unix
    7
    8 # Copy artifacts
    9 # Note: These paths assume you run the build command from the repository root
   10 COPY dockerfiles/sqlserver/init-db.sql /init-db.sql
   11 COPY design/seedData/noteCatalogs.sql /seed-data.sql
   12 COPY src/Hmm.Docker/setup.sql /setup.sql
   13 COPY src/Hmm.Docker/entrypoint.sh /entrypoint.sh
   14
   15 # Fix permissions and line endings
   16 RUN dos2unix /entrypoint.sh && \
   17     chmod +x /entrypoint.sh
   18
   19 USER mssql
   20
   21 ENTRYPOINT ["/entrypoint.sh"]

  Step 4: Create Docker Compose for Testing
  Create src/Hmm.Docker/docker-compose.yml to define the test environment.

  File: src/Hmm.Docker/docker-compose.yml

    1 version: '3.4'
    2
    3 services:
    4   db-test:
    5     build:
    6       context: ../..
    7       dockerfile: src/Hmm.Docker/Dockerfile
    8     environment:
    9       ACCEPT_EULA: "Y"
   10       MSSQL_SA_PASSWORD: "Password123!"
   11       MSSQL_PID: "Express"
   12     ports:
   13       - "14330:1433" # Mapped to 14330 to avoid conflict with local dev DB
   14     networks:
   15       - test-net
   16     healthcheck:
   17       test: ["CMD-SHELL", "/opt/mssql-tools18/bin/sqlcmd -S localhost -U sa -P 'Password123!' -C -Q 'SELECT 1' || exit 1"]
   18       interval: 10s
   19       retries: 5
   20       start_period: 10s
   21
   22   # Optional: Run the API connected to this test DB
   23   api-test:
   24     build:
   25       context: ../..
   26       dockerfile: src/Hmm.ServiceApi/Dockerfile
   27     environment:
   28       - ASPNETCORE_ENVIRONMENT=Development
   29       - ConnectionStrings__HmmNoteConnection=Server=db-test;Database=hmm;User Id=sa;Password=Password123!;TrustServerCertificate=True
   30     ports:
   31       - "8080:80"
   32     depends_on:
   33       db-test:
   34         condition: service_healthy
   35     networks:
   36       - test-net
   37
   38 networks:
   39   test-net:
   40     driver: bridge

  Step 5: Run the Environment
  Navigate to the root directory of your project and run the following command to start the functional test environment:

   1 docker-compose -f src/Hmm.Docker/docker-compose.yml up --build

  Note:
   * The database will be available on localhost:14330.
   * The sa password is set to Password123!.
   * The database name is hmm.
   * The environment is pre-seeded with Catalogs and Authors defined in noteCatalogs.sql.
